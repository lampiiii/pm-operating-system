# Persona Drip

Daily B2B persona immersion system. Generates one vivid, realistic scenario per day from a persona's workday to build PM empathy for roles they've never lived.

## Key Files

- `drip-log.md` — Master log of all drips (date, scenario, category, segment). Check before generating to avoid repeats.
- `drips/` — Individual drip files, named `YYYY-MM-DD-drip-NNN.md`

## How It Works

- Generated by the `persona-drip` custom agent (`.claude/agents/persona-drip.md`)
- Scheduled: weekdays at 11 AM (local cron), daily at 8 AM (remote trigger)
- Default persona: configurable (e.g., **Customer Success Manager**, **Product Manager**, **Sales Engineer**)

## Rotation Rules

**6 categories** (never repeat consecutively): Daily Operations, High-Stakes Moments, Recurring Rituals, Cross-Functional Friction, Strategic Work, Emotional Landscape

**4 segments** (rotate): Enterprise, Commercial, Mid-Market, SMB

## Drip Format

Each drip follows: Scenario (one-line title) > Time > Context > Play-by-Play (8-15 vivid bullets) > Invisible Friction (2-3 product opportunities) > PM Takeaway > Category + Segment + Drip #

## Conventions

- All dates: YYYY-MM-DD
- Drip numbering is sequential across all time
- After generating a drip: save to `drips/`, append row to `drip-log.md`
- Agent memory persists at `~/.claude/agent-memory/persona-drip/`

## Homepage Integration

Drips are displayed on the PM Workspace homepage (`/home`) via the `/api/drips` server endpoint. The homepage shows the latest drip with back/forward navigation to browse older drips, and an expandable view for the full play-by-play, friction, and PM takeaway sections.
